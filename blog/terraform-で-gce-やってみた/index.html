Terraform で GCE やってみた
ad
Nov 8, 2016
1 minute read

GCP が東京リージョン(asia-northeast1)で正式運用開始されたようなので、いい機会だからTerraform を使ってインスタンスを立ち上げたくなった

東京 GCP リージョン の正式運用を開始しました

Terraformのインストール

[@MBP01 workspace]$ brew install terraform
[@MBP01 workspace]$ terraform version  
Terraform v0.7.9

GCPの認証情報を取得

GCP コンソール

  1. API Manager
  2. 認証情報
  3. 認証情報を作成
  4. サービスアカウント キー
  5. Compute Engine default service account
  6. JSON
  7. 作成

Terraform の設定ファイルを準備する

  • サンプル(gce_sample.tf)
# Configure the Google Cloud provider
provider "google" {
   　credentials = "${file(“service_account.json")}"
　　　project = “{project_id}”
    region = "asia-northeast1-a"
}

# Configure the Google Compute Engine Instance
resource "google_compute_instance" "default" {
    name = "test01"
    machine_type = "f1-micro"
    zone = "asia-northeast1-a"
    tags = [“test”]

    disk {
        image = "debian-7-wheezy-v20140814"
    }

    network_interface {
        network = "default"
    }
}

GCE の確認
